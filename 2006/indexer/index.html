The Indexer

Introduction

The Indexer is full-text search engine that is optimized for search speed over very large document collections.

Features

  • Support for partial words, like sp*d (would match speed)
  • Weighted results using term vectors
  • Very simple API
  • Support for specified encoding (or subset of it)

Character Encoding

The Indexer uses character mapper that maps every multibyte character into one byte. This is required for trigram indexing.

Partial matching

For search queries that use wildcards in the words we use the trigram indexing. Every indexed word is splitten into three-byte vectors. The word ID is saved into B-tree in separated file. To get the tree id we use digital tree to store three-byte vectors. Every three-byte vector maps to distinct B-tree. Therefore we can easily find all the words that correspond to the partial word in the query. The words themself are stored in the packed form in sequencial file.
