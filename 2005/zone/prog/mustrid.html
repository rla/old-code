Mustrid programmeerimises

Muster on üldistatud lahendus mingile probleemide klassile.

Mustrite süstemaatika

  1. Programeerimiskeele eripära ja selle poolt pakutavate vahendite kasutamine.
  2. Erilahendus - üldistus puudub, mõeldud on väga konkreetse probleemi lahendust.
  3. Standardlahendus - mingil määral üldistatud lahendus mingit tüüpi probleemile.
  4. Musterlahendus - kindel lahendus antud probleemide klassile.

Tee heade lahenduste leidmiseks

  • Tee üldisemat laadi lahendused võimalikult lihtsaks.
  • Objektidel(isenditel) lase suhelda vaid iseenda, oma väljadega või meetodi atribuutidega.
  • Lahendus on lõplik kui seda ei saa minimalistlikumalt teha.
  • Mitut asja ei pea korraga lahendama.
  • Väldi andmestruktuuride duplikeerimist.

I tüüpi mustrid. Keele võimaluste ärakasutamine

Andmehulkade kandjad (Messenger)

Näide: Kandja Punkt(sisaldab kolme koordinaati) teeb lihtsamaks punkti kasutamise funktsiooni argumendina(vaja läheb ainult ühte argumenti, kui kasutame kandjat Punkt, vastasel juhul vajame kolme argumenti). Kandja, antud juhul punkt, võiks sisaldada ainult piilumismeetodeid. 
class Punkt {
	public int x,y,z;
	public Punkt(int x, int y, int z) {
		this.x=x;
		this.y=y;
		this.z=z;
	}
}

Info korjaja (Collecting parameter)

Kasutatakse funktsiooni argumendina funktsiooni seest info korjamiseks. 
class Korjaja extends ArrayList {}
class MinuKlass {
	public void f(Korjaja k) {
		k.add("esimene");
	}
	public void g(Korjaja k) {
		k.add("teine");
	}
}
