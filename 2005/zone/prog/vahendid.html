Tarkvara arendusvahendid

  • Planeerimisvahendid/meetodid

    • UML (Unified Modeling Language)

      Tarkvaralahenduse struktuuri kirjeldamise vahend. Sobib väga hästi kasutamiseks koos objektorienteeritud lähenemisega. UML skeemidest on võimalik automaatselt koostada vastav OO kood.
      • Ülevaade UML-st
      • UML keele sõnastik
      • UML skeemide koostamise tarkvara - Umbrello - kasutusõpetus

    • RUP (Rational Unified Process)

      Kiiresti arenev tarkvara arenduse põhimõte. Kasutab objektorienteeritud lähenemist ja UML-i. Jagab arendustsükli nelja etappi: eeluuring, kavandamine, valmistamine, lõppviimistlus.
      • Ülevaade RUP-st
      • What is RUP?

  • Koodimisvahendid

    • Eclipse

      Suurepärande arenduskeskkond erinevate programeerimiskeelte tarbeks. Eclipse on kirjutatud Javas ning seetõttu töötab ta erinevatel operatsioonisüsteemidel. Sisaldab tööriistu CVS, UML jt. tehnoloogiate kasutamiseks.
      • phpEclipse - Eclipse PHP laiendus
      • Java graafilise kasutajaliidese disainer

    • ANT

      Tarkvara ehitussüsteem. Asendab unix tüüpi platvormidel make-tüüpi utiile. Kirjutatud Javas, mis tähendab, et ANT töötab väga paljudel operatsioonisüsteemidel.
      • ANT-i build faili visualiseeria
      • Kasutaja interaktsiooni lisamine ANT-i skriptidele
      • ANT-i build failide kasutamine läbi veebilehitseja

  • Testimine

    • JUnit

      Testide koostamise tarkvara Java keelele.

    • phpUnit

      Sarnane JUnit-le, kuid mõeldud keelele PHP.

  • Silumine

    • OProfile

      Linuxi süsteemi profileerija. Võimaldab uurida igasugust parasjagu töötavat baitkoodi.

    • KCachegrind

      Profileerija väljundfailide graafiline vaatleja.

  • Grupitöö

    • phpGroupWare

      Mitmekasutaja-grupitöö organiseermise vahend. Sisaldab kalendrit, aadressiraamatut, projektide haldurit, ToDo listi, vahendit märkmete tegemiseks, veebmaili, failihaldurit jne. Süsteem võimaldab kasutajatele piirangute seadmist, mitmekeelsust, kasutajate eelistusi keskonna suhtes. Üles ehitatud erinevatest moodulitest, mida saab ka ise lisada. Eesti keele toetust (veel) ei ole.

    • Versioonikontroll

      • CVS


        Mitmekülgne süsteem projekti koodifailide erinevate versioonide haldamiseks. Suuremate projektide korral on võimatu tagada kõikide arendajate poolt loodud muudatuste talletamine. Antud veebileht sisaldab vastavat vahendit selle probleemi lahendamiseks. Tegemist on klient-server arhidektuuriga, kus arendajad saadavad oma koodijupid kesksesse hoidlasse. Hiljem on võimalik vaadata erinevusi koodi ridade vahel.
        CVS kasutamist lihtsustavad vahendid:
        • activitymail
          Activitymail on mõeldud versioonikontrollis toimunud muudatustest automaatseks teatamiseks e-maili kaudu.
        • Cervisia
          Lihtne CVS klient, mis tuleb kaasa Linuxi graafilise keskkonna KDE-ga.

      • Subversion

        Alternatiiv CVS-le, väidetavalt parema ja põhjalikuma funktsionaalsusega kui CVS, kuid pole nii palju levinud. Sisaldab suures koguses erinevaid tööd lihtsustavaid lisavahendeid/omadusi, näiteks kaustade ja binaarfailide versioonikontroll. Puuduseks on keeruline võrguarhidektuur.

    • Bugzilla

      Massiliselt levinud online süsteem arendajatele vigadest teatamiseks. Tõstab oluliselt arendustöö produktiivsust.

  • Dokumentatsiooni koostamine

    • Javadoc

      Java klasside ja liideste dokumneerimiseks mõeldud vahend. Dokumentatsioon loetakse välja otse koodist, mis sisaldab vastavaid märgendeid, nn. docblock'e. Koodi seisukohalt on tegemist eriliselt vormindatud kommentaaridega.

    • phpDocumentor

      Töötab sarnaselt Javadoc-ga, kuid on mõeldud PHP jaoks. Lisaks klassidele saab dokumenteerida ka järjestikfaile ja funktsioone. Antud tarkvaraga tulevad kaasa erinevad mallid dokumentatsiooni vormindamiseks(HTML, PDF, XML jpm).
